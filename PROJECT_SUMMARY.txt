════════════════════════════════════════════════════════════════════════════════

through both a Python agent interface and a native Java extension. Enables
intelligent, autonomous security testing operations through conversational
commands and real-time traffic analysis.

QUICK START
════════════════════════════════════════════════════════════════════════════════

1. SETUP
   $ cd /workspaces/omega
   $ ./setup.sh
   (Installs dependencies and creates .env)

2. CONFIGURE
   Edit .env with your Burp Suite connection details:
   - BURP_HOST=localhost
   - BURP_PORT=1337

3. RUN AGENT (Interactive Mode)
   $ python agent.py
   burp-agent> help
   burp-agent> status
   burp-agent> scan https://target.com

4. RUN AGENT SERVER (For Extension)
   $ python agent_server.py
   (Then load Java extension in Burp Suite)

5. BUILD EXTENSION
   $ mvn clean package
   (Creates target/burp-ai-agent-extension-1.0.0.jar)

CONFIGURATION
════════════════════════════════════════════════════════════════════════════════

Environment Variables (.env)
  - BURP_HOST: Burp Suite hostname (default: localhost)
  - BURP_PORT: Burp Suite REST API port (default: 1337)
  - BURP_API_KEY: Optional API key for authentication
  - BURP_USE_HTTPS: Enable HTTPS for Burp connection
  - AGENT_HOST: Agent server hostname (default: localhost)
  - AGENT_PORT: Agent server port (default: 9999)
  - VERBOSE_MODE: Enable verbose logging (default: true)

Maven Configuration (pom.xml)
  - Java version: 11
  - Dependencies: Burp API, GSON, Apache HttpClient
  - Build plugins: Maven Compiler, Shade, JAR

SUPPORT & TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

Connection Issues
  - Verify Burp Suite is running
  - Check BURP_HOST and BURP_PORT in .env
  - Enable REST API in Burp Preferences

Extension Not Loading
  - Rebuild with Maven: mvn clean package
  - Verify Java 11+ is installed
  - Check extension log output

Analysis Not Appearing
  - Ensure agent server is running: python agent_server.py
  - Check extension is enabled in Burp
  - Review Burp output tab for errors

Performance Issues
  - Disable analysis for static assets
  - Reduce traffic volume during testing
  - Monitor CPU/memory usage
